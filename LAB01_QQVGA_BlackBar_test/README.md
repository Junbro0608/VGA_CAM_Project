# LAB01_QQVGA_BlackBar_test

106x120 이미지 바운딩 박스 & 구분선 오버레이 구현 기록.

![Day1 바운딩 박스 결과](./images/day1_result.jpg)

## 요약

- 106x120 이미지 위에 외곽 테두리 + 내부 구분선(상단 메인 / 하단 3분할 썸네일) 오버레이 구현
- 디버그를 위해 테두리 색을 검정 대신 빨간색으로 임시 출력해 위치 육안 확인
- 표시 영역(106x120) 밖으로 선이 뻗어나가는 버그 발견 및 수정
- `ImgROM`을 클럭 동기 레지스터 출력으로 변경 (BRAM 추론 유도)
- `de`(display enable) 미반영 버그 수정

## 모듈 구조

![모듈 블록 다이어그램](./images/day1_blockdiagram.jpg)

| 모듈 | 역할 |
|---|---|
| `vga_decoder` | VGA 타이밍 생성, `de` / `x` / `y` 출력 |
| `ImgROM` | 106x120 이미지 데이터 저장, 클럭 동기 레지스터 출력 |
| `ImgRomReader` | `x/y` 기준 표시 영역 판정 + ROM 데이터 → RGB 변환 |
| `Img_externalBar` | `x/y` 기준 테두리 · 구분선 오버레이 → 최종 RGB 출력 |

데이터 흐름: `vga_decoder` → `ImgRomReader` (+ `ImgROM`) → `Img_externalBar`

## 레이아웃 사양

| 영역 | 범위 | 크기 |
|---|---|---|
| 상단 메인 | y = 0 ~ 79 | 106 x 80 |
| 가로 구분선 | y = 80 | 106 x 1 |
| 하단 컬럼1 | x = 0 ~ 34, y = 81 ~ 119 | 35 x 39 |
| 세로 구분선 1 | x = 35 | 1 x 39 |
| 하단 컬럼2 | x = 36 ~ 69, y = 81 ~ 119 | 34 x 39 |
| 세로 구분선 2 | x = 70 | 1 x 39 |
| 하단 컬럼3 | x = 71 ~ 105, y = 81 ~ 119 | 35 x 39 |

검산: 35+1+34+1+35 = 106 ✓ / 80+1+39 = 120 ✓

## 트러블슈팅

| 이슈 | 원인 | 해결 |
|---|---|---|
| 빨간 선이 화면 전체(가로/세로)로 뻗어나감 | border 판정에 106x120 영역 제한이 없었음 | 모든 border 신호를 표시 영역 조건과 AND |
| 외곽 테두리가 없음 | 내부 구분선만 구현, 프레임 로직 누락 | 이미지 가장자리(x=0,105 / y=0,119) 테두리 추가 |
| `de` 미반영 | blanking 구간 처리 누락 | 표시 영역 계산에 `de` 조건 추가 |

## 검증

- 테두리/구분선을 빨간색으로 임시 출력해 위치 확인 완료 (정식 색상은 검정으로 복원 예정)
- 외곽 테두리 + 가로 구분선(1개) + 세로 구분선(2개)이 106x120 영역 안에서만 정확히 표시됨 확인 (상단 이미지 참고)
