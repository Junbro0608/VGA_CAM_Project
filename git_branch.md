# Git 브랜치 만들고 사용하기 가이드

Git에서 브랜치(Branch)는 원본 코드를 안전하게 유지하면서 새로운 기능을 개발하거나 버그를 테스트하기 위해 만드는 **독립적인 작업 공간**입니다. 마치 하나의 스토리에서 분기되는 '평행 우주'를 만드는 것과 같습니다.

---

### 1. 현재 상태 확인하기
먼저 내가 지금 어떤 브랜치에서 작업 중인지 확인합니다.
```bash
git branch
```
* 목록 중에서 현재 작업 중인 브랜치 앞에는 `*` 기호가 붙어 있습니다.

---

### 2. 새로운 브랜치 만들기
새로운 작업을 위한 브랜치를 생성합니다.
```bash
git branch <새로운-브랜치-이름>
# 예: git branch feature-login
```

---

### 3. 만든 브랜치로 이동하기 (Switch)
작업 공간을 방금 만든 새 브랜치로 변경합니다. (과거에는 `checkout`을 주로 썼지만, 최근에는 역할이 명확하게 분리된 `switch` 명령어 사용을 권장합니다.)
```bash
git switch <새로운-브랜치-이름>
# 예: git switch feature-login
```

> **💡 꿀팁: 한 번에 만들고 바로 이동하기**
> 실무에서 가장 자주 쓰이는 단축 명령어입니다. 브랜치를 생성함과 동시에 해당 브랜치로 즉시 이동합니다.
> ```bash
> git switch -c <새로운-브랜치-이름>
> # 구버전 명령어: git checkout -b <새로운-브랜치-이름>
> ```

---

### 4. 새 브랜치에서 작업 및 커밋
이제 코드를 수정하고 평소처럼 커밋을 진행합니다. 여기서 발생한 변경 사항은 현재 브랜치에만 기록되며, 메인 브랜치(`main` 또는 `master`)에는 전혀 영향을 주지 않습니다.
```bash
git add .
git commit -m "로그인 기능 개발 시작"
```

---

### 5. 메인 브랜치에 결과물 합치기 (Merge)
새 브랜치에서의 작업이 성공적으로 끝났다면, 그 결과물을 다시 메인 브랜치에 합쳐야 합니다.
```bash
# 1. 먼저 기준이 되는 메인 브랜치로 돌아갑니다.
git switch main

# 2. 작업했던 브랜치의 내용을 현재(main) 브랜치로 끌어와서 합칩니다.
git merge <새로운-브랜치-이름>
# 예: git merge feature-login
```

---

### 6. 다 쓴 브랜치 삭제하기 (정리)
성공적으로 병합(Merge)이 완료되어 더 이상 필요 없는 브랜치는 깔끔한 프로젝트 관리를 위해 삭제하는 것이 좋습니다.
```bash
git branch -d <새로운-브랜치-이름>
# 예: git branch -d feature-login
