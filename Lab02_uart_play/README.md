# 🎹 FPGA-PC 연동 실시간 멀티채널 풀 오케스트라 시스템
> **Basys3 FPGA Board & Python Serial/Audio Integration Project**

본 프로젝트는 PC 키보드 입력 인터페이스와 Basys3 FPGA 하드웨어 제어판을 UART 시리얼 통신으로 통합하여, 6개 오케스트라 악기 세션의 실시간 단일음 제어 및 병렬 멀티채널 화음 출력을 지원하는 임베디드 오디오 합성 시스템입니다.

---

## 🛠️ 시스템 아키텍처 및 통신 프로토콜 (Principle)

### 1. UART 8비트 데이터 패킷 명세
FPGA에서 버튼(`btn_r`)을 누르는 순간 래치(Latch)된 스위치 데이터 1바이트(8-bit)가 PC로 전송되며, 하위 5비트는 다음과 같은 디지털 논리로 디코딩됩니다.

* **음계 코드 (sw[1:0] - 2-bit):**
  * `2'b00`: 소리 없음 (Rest)
  * `2'b01`: 도 (Do) / 1선율 (킥 / 크래시)
  * `2'b10`: レ (Re) / 2선율 (스네어 / 스틱)
  * `2'b11`: 미 (Mi) / 3선율 (하이햇 / 초크)
* **악기 코드 (sw[4:2] - 3-bit):**
  * `3'b000`: 피아노 (Piano)
  * `3'b001`: 트럼펫 (Trumpet)
  * `3'b010`: 드럼 (Drum)
  * `3'b011`: 오케스트라 심벌즈 (Cymbals)
  * `3'b100`: Bb 클라리넷 (Clarinet)  *바순에서 교체*
  * `3'b101`: 바이올린 (Violin)

### 2. 하드웨어 스펙 (FPGA RTL 사양)
* **시스템 마스터 클럭:** 100MHz (`sys_clk_pin`, W5 핀 매핑)
* **통신 속도 (Baud Rate):** 115,200 bps
* **리셋 논리:** Active High 비동기 리셋 (`posedge rst`, U18 중앙 버튼 매핑) - Artix-7 FF 최적화 구조
* **채터링 방지:** 20비트 카운터 기반 디지털 디바운싱 회로 내장 (약 10.4ms 필터링) 및 라이징 엣지 트리거 회로 구동 (`btn_r`, T17 핀 매핑)

## 🕹️ 병렬 제어 인터페이스 레이아웃 (Application)

PC 키보드로 수동 연주를 하거나 FPGA 보드의 스위치 조작 후 버튼을 누르면 해당 악기 전용 가상 오디오 채널(Channel 0~5)이 깨어나며 음원을 실시간으로 합성(Mixing)합니다.

| 악기 라인 | 채널 ID | 1음 (도 / 킥 / 큰울림) | 2음 (레 / 스네어 / 작은울림) | 3음 (미 / 하이햇 / 숏타격) |
| :--- | :---: | :---: | :---: | :---: |
| **피아노 (Piano)** | Ch 0 | **Q** | **W** | **E** |
| **트럼펫 (Trumpet)** | Ch 1 | **A** | **S** | **D** |
| **드럼 (Drum)** | Ch 2 | **Z** | **X** | **C** |
| **심벌즈 (Cymbals)** | Ch 3 | **I** | **O** | **P** |
| **클라리넷 (Clarinet)** | Ch 4 | **K** | **L** | **;** |
| **바이올린 (Violin)** | Ch 5 | **,** (쉼표) | **.** (마침표) | **/** (슬래시) |
